# Format: Government / Public-Sector / Competency-Based

For government and similar public-sector roles. These are screened differently
from tech jobs: a panel scores explicit **competencies**, HR verifies
**titles, dates, and hours**, and equivalency clauses matter. Honesty and
completeness beat flash.

## What's different from a tech resume
- **Hours per week** are often required on every role — public-service hiring weights full-time vs part-time toward the "X years of related experience" equivalency. Put `Month Year – Month Year · Full-time, NN hrs/week` (or `Part-time, NN hrs/week`) on each entry. Pull hours from `MASTER-PROFILE.md`; if missing, ask — don't guess.
- **Concurrent roles**: if two part-time roles overlapped, say so in the description ("Held concurrently with X, 50% each of NN hr/week standard") so the timeline isn't read as impossibly additive.
- **Official titles** on every entry (HR verifies). The elevated tagline lives only under the name.
- **Full work history** — include older roles (condensed is fine), since equivalency is computed from total related experience.
- **Residency / location** — if the posting requires it, state the city/region in the header and summary.
- **Education equivalency** — if degrees aren't an exact match, invoke the posting's "related field" / experience-based equivalency clause, but place it *after* you've led with strengths, not as an opening apology.

## Core Competencies section (the scored part)
Add a **"Core Competencies"** section near the top with one concise evidence
line per competency **the posting names as critical**. Use the posting's exact
list — public-service frameworks differ, so match what's asked rather than a
generic set. Each line = competency name + a concrete, real example from the
profile. (Typical competencies include things like adaptability, collaboration,
systems thinking, problem solving, and developing self/others — but always
defer to the posting's own wording.)

## Honest hedges that fit public-sector scrutiny
- A standard you meet via an equivalent regime → "X-equivalent ... discipline" rather than claiming the certification (panels respect the accuracy).
- A tool/stack you don't run daily → a "Familiar / Ramping" skills line: the underlying concepts are daily work, the specific tooling is a fast ramp.
- Tenure → split honestly ("N years in the field; the last M on the specific platform/specialty").

## Resume order
Header → Summary → Core Competencies → Professional Experience (with hours) →
Earlier Experience (with hours) → Technical Skills → Education. US Letter (or
the local standard), 2 pages where possible (public-service applications
tolerate longer, but tight is better).

## Cover letter (1 page)
Calm, evidence-first, public-sector register. Open by naming the requisition /
competition and leading with the strongest proof point (not a comparison to
other applicants). Cover the core of the role, the role-specific specifics,
reliability/process discipline, stakeholder support, and any differentiators.
Handle the education-equivalency note briefly and late. Close on the named
competencies. Reference the requisition / competition number.

## STAR competency stories (interview prep — generate for these roles)
Produce a separate `<name>-<company>-STAR` doc (PDF + source): one
Situation–Task–Action–Result story per named competency, each ~60–90 seconds
spoken, drawn from real experience in the profile. Add a short "JD
responsibility → which story" mapping and prepped answers for likely probes
(e.g. a non-exact-match degree; depth on a tool framed as "ramping"). This is
interview prep, not submitted with the application.
