# Format: Government of Alberta / Public-Sector / APS Competency-Based

For Government of Alberta and similar public-sector roles. These are screened
differently from tech jobs: a panel scores explicit **competencies**, HR
verifies **titles, dates, and hours**, and equivalency clauses matter. Honesty
and completeness beat flash.

## What's different from a tech resume
- **Hours per week** are required on every role — GoA weights FT vs PT toward the "X years of related experience" equivalency. Put `Month Year – Month Year · Full-time, NN hrs/week` (or `Part-time, NN hrs/week`) on each entry. Pull hours from `MASTER-PROFILE.md`; if missing, ask — don't guess.
- **Concurrent roles**: if two part-time roles overlapped, say so in the description ("Held concurrently with X, 50% each of NN hr/week standard") so the timeline isn't read as impossibly additive.
- **Official titles** on every entry (HR verifies). The elevated tagline lives only under the name.
- **Full work history** — include older roles (condensed is fine), since equivalency is computed from total related experience.
- **Residency** — if the posting requires it, state the city/region in the header and summary.
- **Education equivalency** — if degrees aren't an exact match, invoke the posting's "related field" / experience-based equivalency clause, but place it *after* you've led with strengths, not as an opening apology.

## Core Competencies section (the scored part)
Add a **"Core Competencies (APS Model)"** section near the top with one concise
evidence line per competency **the posting names as critical** (use the
posting's list, not the full 7-competency model — match exactly what's asked).
The common APS five: Agility, Develop Self & Others, Build Collaborative
Environments, Systems Thinking, Creative Problem Solving. Each line = competency
name + a concrete, real example from the profile.

## Honest hedges that fit public-sector scrutiny
- ITIL → "ITIL-equivalent service-management discipline under accreditation" (panels respect the accuracy).
- Microsoft stack you don't run daily (IIS/AppViewX/.NET) → a "Familiar / Ramping" skills line: concepts are daily work, the specific tooling is a fast ramp.
- Tenure → split honestly ("20+ years in production infrastructure; the last N on the specific platform").

## Resume order (GoA)
Header → Summary → Core Competencies (APS Model) → Professional Experience
(with hours) → Earlier Experience (with hours) → Technical Skills → Education.
US Letter, 2 pages where possible (GoA tolerates longer, but tight is better).

## Cover letter (1 page)
Calm, evidence-first, public-sector register. Open by naming the requisition
and leading with the strongest proof point (not a comparison to other
applicants). Cover: the core of the role, the cloud/ops specifics, reliability
& change discipline, stakeholder support, and AI where relevant. Handle the
education-equivalency note briefly and late. Close on the named competencies.
Reference the requisition number.

## STAR competency stories (interview prep — generate for these roles)
Produce a separate `<name>-<role>-STAR` doc (PDF + source): one
Situation–Task–Action–Result story per named competency, each ~60–90 seconds
spoken, drawn from real experience in the profile. Add a short "JD
responsibility → which story" mapping and prepped answers for likely probes
(e.g., non-CS degree; depth on a tool framed as "ramping"). This is interview
prep, not submitted with the application.
